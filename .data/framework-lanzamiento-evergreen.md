# Framework de Lanzamiento Evergreen por Instagram Stories

> **Qué es:** Sistema de venta permanente por Instagram Stories que funciona sin lanzamientos puntuales, sin ads obligatorios, y sin webinars.
> **Para quién:** Cualquier creador/marca con producto digital y audiencia en IG. Aplicable a ADP (Jesús) y otros clientes.
> **Fuentes cruzadas:** 192 piezas de Nico (alumno de Nafe, 2.5 meses evergreen) + 90 guiones de Nafe (StoryMakers) + Framework Turu (560 stories) + Framework Ego (50 stories) + 88 referencias analizadas + 562 compradores reales ADP
> **Fecha:** 2026-03-31

---

## PARTE 1: LA FILOSOFÍA

### Por qué stories y no otro canal

**Dato fundacional (Nafe + Nico lo confirman):** "El 80% de las ventas en Instagram vienen de stories." Los reels son para ALCANCE (nuevos seguidores). Las stories son para CONVERSIÓN (seguidores → leads → ventas). Son 2 sistemas con objetivos diferentes.

**Por qué funciona para evergreen:**
- Las stories desaparecen en 24h → urgencia NATURAL (no artificial)
- Quien ve tus stories YA te sigue → audiencia CALIENTE
- El DM es el canal de respuesta → conversación 1 a 1 (setter puede intervenir)
- No requiere inversión en ads → margen 100%
- Se puede reciclar → producción sostenible a largo plazo

### Los 3 principios del evergreen (extraídos de Nico + Nafe + Turu)

**Principio 1: MODULARIDAD** (de Nico)
La secuencia de venta tiene partes FIJAS (cierre, CTA, lead magnet) y partes VARIABLES (hook, caso, analogía). Las fijas se crean UNA VEZ y se reciclan. Las variables se rotan. Producir una secuencia nueva = 5 minutos (cambiar el hook).

**Principio 2: CICLO INSPIRAR-EXPIRAR** (de Nico, confirmado por Nafe)
NO alternar personalidad-venta-personalidad-venta cada día. Hacer RACHAS de 3-4 días del mismo tipo. La personalidad PRECARGA la confianza que la venta GASTA. Sin precarga = spam. Sin venta = solo entertainment.

**Principio 3: INVERSIÓN DE ROLES** (de Turu, confirmado por Nafe M3)
NUNCA vender. Que la audiencia te lo pida. Mostrar demanda ("me llegan mensajes todo el día") → "ceder" ("bueno, ya que insisten"). El comprador siente que GANÓ acceso, no que le vendieron.

## PARTE 2: LA ARQUITECTURA

### 2.1 Los 3 tracks del sistema

```
TRACK A: PERSONALIDAD (confianza)
   → Zero CTA. Puro. 100% emoción/conexión/valores.
   → Frecuencia: 3-4 días consecutivos por ciclo.
   → Objetivo: que el viewer piense "me cae bien, confío".

TRACK B: EVERGREEN (conversión)
   → CTA con keyword. Template modular.
   → Frecuencia: 2-3 días consecutivos por ciclo.
   → Objetivo: que el viewer responda la keyword.

TRACK C: PRUEBA SOCIAL (credibilidad)
   → Casos de alumnos/compradores en 5 formatos.
   → Frecuencia: intercalado con A y B.
   → Objetivo: que el viewer piense "funciona para gente como yo".
```

### 2.2 El template evergreen modular (de Nico)

```
┌─────────────────────────────────────────────────┐
│  ZONA VARIABLE (cambia cada vez)                │
│                                                  │
│  S1: HOOK rotativo                              │
│      → Resultado / Gamification / DM /           │
│        Analogía IA / Meme / Claim               │
│                                                  │
│  S2: PRUEBA del hook                            │
│      → Comprobante / Matemática / Caso /         │
│        Screenshot                                │
│                                                  │
│  S3: CASO de alumno/comprador (opcional)         │
│      → Foto + @ + métrica                        │
├─────────────────────────────────────────────────┤
│  ZONA FIJA (se crea 1 vez, se recicla)          │
│                                                  │
│  S4: MECANISMO (1 frase simple)                 │
│      → "La clave? [método]. Nada más."          │
│                                                  │
│  S5: LEAD MAGNET (mostrar por dentro)           │
│      → Screenshot del recurso ABIERTO           │
│                                                  │
│  S6: CTA (keyword GRANDE, color diferente)      │
│      → "Respondé [KEYWORD] y te lo mando"       │
│      → Foto producida (única en la secuencia)   │
└─────────────────────────────────────────────────┘
```

**Reglas del template:**
1. **S4-S5-S6 son ASSETS PERMANENTES.** Se fotografían/diseñan UNA VEZ. Se reutilizan durante meses.
2. **S1-S2 se cambian cada secuencia.** Cada semana (o cada ciclo) se usa un hook diferente.
3. **S3 es OPCIONAL.** Si hay un caso nuevo, se inserta. Si no, se salta.
4. **El lead magnet se muestra POR DENTRO** (A1 verificable). Nunca decir "te mando un PDF". Mostrar el documento abierto con títulos/links visibles.
5. **La foto de CTA es la ÚNICA foto producida** de toda la secuencia. El contraste visual señaliza "esto es importante".
6. **La keyword para la oferta principal es UNIVERSAL** (ej: "YO", "CLASE"). Las keywords para lead magnets específicos son TEMÁTICAS (ej: "GANCHOS", "PROMPT").

### 2.4 Rotación de lead magnets (7 tipos)

| # | Tipo | Ejemplo | Keyword | Duración |
|---|---|---|---|---|
| 1 | **Doc con contenido real** (DEFAULT) | Google Drive con templates/secuencias/reels | CLASE / YO | Permanente — se usa 1 de cada 3 veces |
| 2 | **Micro recurso accionable** | 5 prompts, checklist, plantilla de 1 página | Temática (PROMPT, GANCHOS) | 2-3 semanas |
| 3 | **Fragmento del producto real** | Grabación de 1 clase real de la formación | YO / CLASE | 1 mes |
| 4 | **Pieza única de curiosidad** | "La story que le hizo ganar $250K" (borrosa → reveal) | YO | 1-2 semanas |
| 5 | **Video corto explicativo** | "5 min explicando cómo funciona" | YO | 1 mes |
| 6 | **Asesoría gratuita** (escasez real) | "Esta semana regalo 5 asesorías" | YO | 1 semana |
| 7 | **Acceso a comunidad/grupo** | "Te agrego al grupo de WhatsApp" | COMUNIDAD | Permanente |

**Regla:** El LM DEFAULT (#1) se usa como cierre fijo. Los LM temáticos (#2-#7) se rotan como zona variable. Nunca ofrecer el mismo LM temático 3 semanas seguidas.

### 2.5 Los 5 formatos de prueba social (espectro formal→invisible)

```
MÁS FORMAL                                                    MÁS INVISIBLE
     |                                                              |
Formato C          Formato B          Formato A       Incidental      Caballo de Troya
(masivo IA)        (historia          (corto           (pago durante   (contenido emocional
 7+ casos)          completa)          3 stories)       actividad)      → resultado al final)
     |                |                  |                 |                  |
guardia ALTA      guardia MEDIA      guardia MEDIA    guardia BAJA     guardia ZERO
mensual           quincenal          semanal          2x/semana        mensual
```

**Principio:** La potencia es INVERSAMENTE proporcional a la visibilidad. Usar los 5 niveles en paralelo. Los formatos invisibles (incidental + Troya) son los más potentes pero requieren momentos reales.

### 2.6 Cadencia (ciclo inspirar-expirar)

**NO es un calendario fijo.** Nico no sigue "Lunes=P, Martes=E". Opera por RACHAS orgánicas — a veces 4 días P seguidos porque viajó con la familia, a veces 3 días E seguidos porque tiene casos frescos. La cadencia es un RITMO, no un cronograma.

**El ritmo:**
```
INSPIRAR (3-4 días P) → EXPIRAR (2-3 días E) → INSPIRAR → EXPIRAR...
```

**Reglas del ritmo:**
1. **NUNCA más de 4 días E consecutivos.** La audiencia se agota.
2. **NUNCA más de 4 días P consecutivos.** Se pierde conversión.
3. **Los P SIEMPRE preceden a los E más agresivos.** La racha P precarga la racha E.
4. **Los pares P→E son ESTRATÉGICOS** (H44b): el tipo de P del último día prepara al tipo de E del primer día.
5. **Cada 4-6 semanas: SEMANA ANTI-CTA** (de Nafe). Zero CTA durante 5-7 días. Solo mostrar DMs de gente pidiendo. "Mucha gente me escribe preguntando..." sin ofrecer NADA. Genera demanda reprimida que EXPLOTA cuando vuelve el CTA.
6. **Ratio orientativo:** 55-65% E / 35-45% P. Ajustar por audiencia: más joven → más E, más adulta → más P.

### 2.7 La semana Anti-CTA (de Nafe — demanda reprimida)

**Qué es:** Cada 4-6 semanas, PARAR completamente de vender. Ni CTA, ni keyword, ni link. Solo:
- Mostrar screenshots de DMs: "¿Cuándo abrís de nuevo?", "¿Cómo me anoto?"
- Responder con evasión: "No sé si voy a hacer algo", "Déjenme en paz 😂"
- Mostrar números de demanda: "Me escribieron +200 personas esta semana"
- NO ofrecer reapertura

**Por qué funciona:**
- Nafe M3: "La escasez es una posición ética, no una táctica"
- La AUSENCIA de CTA después de semanas de CTA genera CONTRASTE
- El viewer que venía viendo CTAs sin responder ahora piensa "mierda, se cerró"
- Cuando el CTA vuelve la semana siguiente, las respuestas son 2-3x más altas

**Nafe lo usa en el DÍA +8 post-lanzamiento:** "Podría ganar +100K euros abriendo sin filtros. Pero eso NO va a pasar." + capturas de gente rogando. Zero CTA. En evergreen, se traduce en 1 semana cada 4-6 de "demanda reprimida".

### 2.8 Stories H48 — 2 contenidos completos en 1 contenedor (de Javier @javierroddom)

> Fuente: 6 instancias verificadas visualmente en 288 stories evergreen (oct-dic 2025). Análisis completo en `.data/analisis-creador-dubai-H48-tension-capas.md`.

**Qué es:** En stories de TH largo (30-60s), el texto overlay funciona como POSTER MOTIVACIONAL autosuficiente (5-10 palabras) y el audio funciona como CONFESIÓN PERSONAL completa e independiente. Son 2 productos editoriales diferentes en 1 contenedor. El viewer sin sonido recibe versión A completa. El viewer con sonido recibe versión B completa.

**Por qué importa para el evergreen:** Las stories H48 son los momentos de PROFUNDIDAD del Track A (personalidad). Sin ellos, los días P son lifestyle sin conexión emocional. Con ellos, el viewer que activa el sonido CONOCE al creador íntimamente = la confianza construida es más fuerte = la conversión de los días E es más alta.

**Cuándo usar:** Solo en Track A (personalidad). NUNCA en Track B (evergreen) ni Track C (prueba social). 2 stories H48 por semana máximo (~1.5/semana es la cadencia de Javier).

**Cómo producir (5 min por story):**
1. Grabar TH de 30-45s hablando libremente (sin guión) en contexto cotidiano
2. Escuchar y elegir 1 frase de 5-10 palabras como poster overlay
3. Publicar SIN subtítulos

**Regla crítica: H48 es INCOMPATIBLE con subtítulos karaoke (§8.4 de Naffe "overlay ≠ hablado" llevado al extremo).** Si ponés subtítulos, destruís la independencia del poster como producto completo. Regla: stories de venta (Track B) → subtítulos SIEMPRE. Stories H48 de personalidad (Track A) → sin subtítulos.

**3 variantes (de más a menos potente):**
- **Contradicción triple:** visual=status + texto=anti-materialismo + audio=vulnerabilidad. La más fuerte: 3 mensajes opuestos generan tensión cognitiva
- **Contradicción doble:** texto=frase casual + audio=historia profunda diferente
- **Expansión alineada:** texto y audio sobre el mismo tema, audio profundiza. La más suave

**Banco de combinaciones (para producción masiva):** 10 audios grabados en 1 sesión × 10 textos overlay = 100 combinaciones = 50 semanas sin repetir.

| # | Texto overlay (poster) | Tema del audio (confesión) |
|---|----------------------|---------------------------|
| 1 | "La mayoría piensa que necesitás plata para empezar" | Cómo empezó con $20 y celular |
| 2 | "Nadie te enseña esto" | Primer fracaso digital |
| 3 | "Lo que me hubiera gustado saber a los 30" | Historia del origen (herrería/changas) |
| 4 | "Mi viejo me enseñó algo" | Lección de la herrería → emprendimiento |
| 5 | "Esto me da bronca" | Anti-gurús (H49 verbal adaptado) |
| 6 | "No es talento, es sistema" | Historia de alumno mayor con primera venta |
| 7 | "Si tuviera que empezar de cero" | Los 3 pasos del proceso |
| 8 | "Mi pareja me dijo que estaba loco" | Historia real de cuando dejó lo anterior |
| 9 | "El mensaje que más me emocionó" | WA de alumno con primera venta |
| 10 | "Hay algo que no cuento nunca" | Momento vulnerable real |

## PARTE 3: EL SISTEMA DE PRODUCCIÓN

### 3.1 Assets a crear UNA VEZ

**Banco de 10 fotos anchor (personalidad):**
1. Foto antigua / de la infancia / del origen
2. Foto con familia (pareja, hijos, padres)
3. Foto viajando / lifestyle actual
4. Foto trabajando (setup, laptop, celular)
5. Foto con equipo / calls / behind the scenes
6. Foto con comprador/alumno
7. Screenshot WA del comprador más representativo
8. Foto del producto por dentro (plataforma, módulos)
9. Foto "CTA" producida (fondo producido, iluminación)
10. Foto "pasado" para contraste

**Cada foto se usa 1x/mes con texto diferente.** 10 fotos × 4 textos = 40 stories de personalidad (10 semanas sin repetir).

**Assets fijos del template evergreen:**
- Foto S4 (mecanismo): selfie casual + frase overlay
- Foto S5 (lead magnet): screenshot del recurso POR DENTRO
- Foto S6 (CTA): foto producida + keyword GRANDE
- Estos 3 se reciclan durante MESES. Solo se actualizan si el lead magnet cambia.

### 3.2 Workflow semanal (producción)

```
DOMINGO (30 min):
  1. Elegir hook de la semana (de banco de 10 tipos)
  2. Elegir lead magnet de la semana (de banco de 7 tipos)
  3. Elegir caso de alumno de la semana (de banco de compradores)
  4. Producir S1-S2 nuevos (1 foto o video + texto overlay)
  5. Verificar que S4-S6 siguen vigentes

DURANTE LA SEMANA:
  - Publicar según cadencia
  - En días P: grabar 1-2 THs H48 (30-45s hablando libre + poster overlay después)
  - Capturar momentos reales para:
    → Pago incidental (notificación durante actividad)
    → Caballo de Troya (secuencia con equipo + "por cierto")
    → Screenshots WA nuevos de compradores
  - Responder DMs o derivar a setter

VIERNES (15 min):
  - Seleccionar los mejores DMs recibidos para prueba social
  - Anotar qué hook funcionó mejor (para reciclar después)
  - Preparar el hook de la semana siguiente
```

### 3.3 Producción masiva de prueba social con IA

**Para Formato C (mensual):**

1. Seleccionar 7 compradores de nichos DIFERENTES
2. Para cada uno generar fondo IA del nicho:
   ```
   Prompt: "Ilustración estilo anime/comic de [persona en su nicho:
   cocina/yoga/oficina/taller/consultorio]. Estilo cálido,
   profesional, fondo detallado del entorno de trabajo."
   ```
3. Overlay: screenshot WA + perfil si tiene + métrica grande
4. Tiempo total: ~30 minutos para 7 casos
5. Resultado: 7 stories de prueba social que parecen ÚNICAS

### 3.4 Bloques modulares insertables (de Nafe)

Pre-producir 2 tipos de bloque que se INSERTAN en cualquier secuencia:

**BLOQUE PRUEBA SOCIAL (↑↑):**
```
2-3 stories pre-hechas:
  - Collage de 6-9 screenshots WA de compradores
  - Foto de call grupal con alumnos (Zoom)
  - Dato: "+562 personas ya lo hicieron" (o el número real)
```
Se insertan DESPUÉS de la zona variable (S2-S3) y ANTES de la zona fija (S4-S6). Agregan credibilidad cuando el hook es débil o nuevo. Se reciclan actualizando los screenshots cada 2-3 meses.

**BLOQUE PUSH (↓↓):**
```
1-2 stories pre-hechas:
  - "Solo esta semana" o "Las plazas se llenan rápido"
  - Screenshot de DMs pidiendo + "ya casi no doy abasto"
```
Se insertan DESPUÉS del CTA (S6) como refuerzo. Solo usar en semanas de venta agresiva. NO usar más de 1 vez cada 2 semanas para no quemar.

**Combinación:** Narrativa (70%) → [BLOQUE PRUEBA SOCIAL] → CTA → [BLOQUE PUSH]

### 3.5 Pausa de 15-20 minutos entre stories (de Nafe)

**Qué es:** No publicar todas las stories de golpe. Separar por bloques con pausas.

**Cómo funciona:**
1. Publicar stories S1-S2 (hook + prueba)
2. **ESPERAR 15-20 minutos**
3. Las reacciones al hook se ACUMULAN y son visibles para los que llegan tarde
4. Publicar stories S3-S6 (caso + cierre + CTA)

**Por qué funciona:**
- Las reacciones acumuladas en S1-S2 funcionan como PRUEBA SOCIAL ORGÁNICA para los viewers que llegan con la segunda tanda
- El algoritmo re-posiciona la burbuja de stories al frente cuando se agrega contenido nuevo
- El creador aparece 2 veces en el feed del viewer sin crear más contenido

**Cuándo usar:** En secuencias evergreen (Track B). NO en personalidad (Track A) — la personalidad se publica cuando surge, sin estrategia de timing.

### 3.6 Reciclaje (sistema anti-agotamiento)

| Asset | Se recicla cada | Cómo |
|---|---|---|
| Foto de hook gamification | 5-6 semanas | MISMA foto + MISMO texto (nuevos seguidores no la vieron) |
| Foto anchor de personalidad | 4 semanas | MISMA foto + TEXTO diferente |
| Template cierre (S4-S6) | Indefinido | Variación mínima (fondo diferente, mismo texto) |
| Caso de alumno | 8 semanas | MISMO alumno, dato ACTUALIZADO |
| Lead magnet doc | Indefinido | El doc se actualiza pero la STORY se recicla |
| Imagen IA por nicho | 12 semanas | Regenerar con prompt similar |

## PARTE 4: PSICOLOGÍA DEL SISTEMA

### 4.1 Cadena de confianza del sistema evergreen

El viewer nuevo atraviesa FASES de confianza. No hay timeline fijo — cada persona avanza a su ritmo. Lo que importa es el ORDEN de los SÍes internos:

```
FASE 1 — "¿Quién es este?"
  Ve stories P → "Me cae bien" (Axioma 4)
  Ve stories E → "Ah, vende algo" (Axioma 3)
  No responde CTA → NORMAL. Está observando.

FASE 2 — "¿Funciona de verdad?"
  Ve más P → "Es auténtico" (Axioma 1)
  Ve caso alumno → "Funciona para otros" (Axioma 6)
  Ve CTA → CONSIDERA responder. Evalúa.

FASE 3 — "¿Es para mí?"
  Ve caso de alguien parecido → "Si esa persona pudo..." (Axioma 6)
  Ve hook que le resuena → "ESTO me habla a mí"
  Responde keyword → LEAD capturado.

FASE 4 — "¿Puedo confiar mi dinero?"
  Consume lead magnet → "Sabe de lo que habla" (Axioma 1)
  Setter/bot contacta → evaluación personal
  Compra → CLIENTE
```

**La ventaja clave del evergreen:** Cada viewer entra cuando entra. No se pierde nada porque el contenido es CÍCLICO. El viewer que llegó hoy y el que llegó hace 2 meses ven el MISMO tipo de contenido (porque los hooks rotan y se reciclan). No hay "llegaste tarde al lanzamiento".

### 4.2 Las 3 capas de incompletitud (de Nafe, ejecutadas por Nico)

| Capa | Qué da gratis | Qué retiene | Dónde se entrega lo retenido |
|---|---|---|---|
| **Capa 1: La emoción** | 100% — sentir la transformación | Nada | Stories (P + casos) |
| **Capa 2: El qué** | 100% — qué funciona, qué resultados, qué mecanismo | Nada | Stories (E + mecanismo) |
| **Capa 3: El cómo** | 30% — fragmento del método (lead magnet) | 70% — implementación personalizada | Lead magnet → producto pago |

**Regla:** Dar el 100% de emoción y de "qué" gratis. Retener el 100% del "cómo implementar". El lead magnet es un FRAGMENTO del "cómo" que genera sed del resto.

### 4.3 La inversión de roles en modo evergreen (de Turu, adaptada)

En un lanzamiento: 4 semanas de "la audiencia pide" → 1 semana de "el creador cede".
En evergreen: la inversión es PERMANENTE y SUTIL:

| Cómo se manifiesta en evergreen | Ejemplo |
|---|---|
| "Me llegan estos mensajes" (mostrar DMs) | Screenshot WA de compradores "¿cuándo abrís de nuevo?" |
| "Bueno ya que insisten" (no como lanzamiento sino como actitud) | "Mucha gente me pregunta cómo. Armé esto para ustedes" |
| "No es para todos" (filtro de auto-selección) | "Si tenés un conocimiento y querés venderlo, esto es para vos" |
| "Se me llena rápido" (escasez real en asesorías) | "Esta semana regalo 5 asesorías. La semana que viene no sé si puedo" |
| "Nunca presionar" (el CTA es invitación) | "Respondé YO y te lo mando. Abrazo." |


## PARTE 6: FRAMEWORK UNIVERSAL (para otros clientes)

### 6.1 Checklist de setup (antes de arrancar)

```
□ 1. PRODUCTO: ¿Qué vende? ¿A qué precio? ¿Hay versión gratuita/micro?
□ 2. AUDIENCIA: ¿Quién lo sigue? Edad, género, trigger principal.
□ 3. LEAD MAGNET: ¿Qué puede regalar que sea un FRAGMENTO del producto?
□ 4. KEYWORD: ¿Cuál es la keyword principal? (universal: "YO". Temática según LM)
□ 5. MECANISMO: ¿En 1 frase, cómo funciona? "La clave es [X]. Nada más."
□ 6. ASSETS BASE: ¿Tiene las 10 fotos anchor? Si no, sesión de fotos.
□ 7. PRUEBA SOCIAL: ¿Cuántos clientes/compradores reales tiene? ¿Screenshots?
□ 8. SETTER/BOT: ¿Quién responde los DMs? ¿Hay automatización?
□ 9. EMBUDO POSTERIOR: ¿Qué pasa después del DM? ¿Clase? ¿Llamada? ¿Link?
□ 10. TONO: ¿Cómo habla el creador? ¿Formal? ¿Casual? ¿Humor? ¿Empático?
```

### 6.2 Ajuste por tipo de audiencia

| Audiencia | Stories/día | Ratio P/E | Tono | Lifestyle |
|---|---|---|---|---|
| Jóvenes 18-25 | 8-12 | 40/60 | Energético, memes, humor | Viajes, gadgets, gym |
| Profesionales 25-40 | 5-8 | 50/50 | Directo, data, resultados | Setup, laptop, cafetería |
| Mujeres/padres 35-55 | 4-7 | 55/45 | Empático, cercano, austero | Familia, tiempo libre, hijos |
| +55 escépticos | 3-5 | 60/40 | Paciente, demostrativo | Seguridad, paso a paso |

### 6.3 Ajuste por volumen de prueba social

| Compradores | Formato C | Formato B | Frecuencia |
|---|---|---|---|
| 0-10 | No usar | 1 caso expandido/mes | Caso A semanal |
| 10-50 | 3-4 con IA/mes | 1 caso expandido/quincenal | Caso A semanal + incidental |
| 50-200 | 5-7 con IA/mes | 2 casos expandidos/mes | Todos los formatos |
| 200+ | 7+ con IA rotativos | 1 caso expandido/semana | Los 5 formatos en paralelo |

### 6.4 Las 7 reglas inmutables del evergreen (destiladas de todo el análisis)

1. **El cierre se crea UNA VEZ y se recicla.** Solo el hook cambia.
2. **Los días P y E nunca se mezclan.** Cada día es 100% uno o 100% otro.
3. **El lead magnet se muestra POR DENTRO.** Nunca decir "te mando algo". Mostrar el documento/clase/recurso abierto.
4. **La keyword para la oferta principal es UNIVERSAL.** Las temáticas se reservan para lead magnets específicos.
5. **Las rachas son de 3-4 días.** No alternar P-E diariamente. Hacer bloques. El ritmo es ORGÁNICO, no un calendario rígido.
6. **Los pares P→E son estratégicos.** El tipo de personalidad prepara al tipo de venta que sigue.
7. **La prueba social más potente es la más INVISIBLE.** Priorizar incidental y Caballo de Troya sobre casos formales.
8. **Posición del CTA — 2 escuelas:**
   - **Escuela Nico:** CTA en ÚLTIMA story, 1 sola vez. Más eficiente. Funciona con audiencia joven.
   - **Escuela Nafe:** CTA en PENÚLTIMA (video hablando) + refuerzo en ÚLTIMA (foto + keyword). CTA se repite 2x. Funciona con audiencia que necesita repetición (35+).
   - Probar Nafe primero. Si respuestas son altas, simplificar a Nico.
9. **Cada 4-6 semanas: semana Anti-CTA.** Zero venta. Solo mostrar DMs pidiendo sin ofrecer. Demanda reprimida → cuando vuelve el CTA, respuestas 2-3x.
10. **Pausa 15-20 min entre bloques de stories.** Publicar hook → esperar → publicar cierre. Las reacciones al hook son prueba social orgánica para el cierre.

## PARTE 7: ERRORES A EVITAR

1. **Hacer todo nuevo cada semana.** El reciclaje NO es pereza — es sistema. Las fotos buenas se reutilizan.
2. **Mezclar P y E en la misma secuencia.** "Te cuento mi historia y por cierto respondé CLASE" contamina ambos.
3. **Poner CTA en cada story.** El CTA va SOLO en la zona fija (S4-S6). El resto es hook y prueba.
4. **Usar urgencia artificial.** "Últimas 3 plazas" cuando no es verdad destruye confianza. La urgencia natural de las stories (24h) es suficiente.
5. **Copiar el tono de otro creador.** El sistema es transferible, el TONO no. Jesús habla como Jesús, no como Nico.
6. **Empezar con Formato C sin tener Formato A probado.** La progresión es A → B → C en meses.
7. **Publicar 15 stories/día para audiencia 35+.** Menos es más. 5-7 bien hechas > 15 mediocres.
8. **Olvidar los pares P→E.** La personalidad aleatorizada no precarga nada. La personalidad PAREADA precarga específicamente.
9. **No capturar momentos reales.** Los pagos incidentales y Caballos de Troya requieren VIVIR con la cámara lista. No se fabrican.
10. **Medir solo ventas.** Las métricas intermedias (views, DMs, responses) son leading indicators. Si bajan, la venta baja después.
11. **Poner subtítulos en stories H48.** Si ponés karaoke a un TH de personalidad con poster overlay, destruís la independencia del poster. Subtítulos = Track B (venta). Sin subtítulos = Track A con H48.

## PARTE 8: PLAYBOOK DE NAFFE (de 88 guiones analizados)

> Las partes 1-7 definen el SISTEMA. Esta parte agrega el PLAYBOOK — las herramientas concretas de ejecución extraídas de analizar 88 guiones de Naffe slide por slide. Detalle completo en `.data/analisis-naffe-master.md`.

### 8.1 Los 8 modelos mentales de Naffe

1. **La Asimetría Informacional como Moneda** — lo que no decís genera deseo
2. **La Historia Personal es el Único Argumento de Venta** — 6/10 guiones son historia SIN CTA
3. **La Escasez es una Posición Ética, No una Táctica** — rechaza dinero real por convicción
4. **El Entretenimiento es el Vehículo, No la Distracción** — la diversión ES la venta
5. **La Prueba Social se Muestra, No se Argumenta** — nunca dice "soy exitoso", muestra
6. **El Template es Invisible, la Variación es Visible** — ~15 templates, 3-4 variables
7. **La Vulnerabilidad es Estratégica, No Terapéutica** — dolor pasado con propósito, nunca sin resolver
8. **El Algoritmo se Hackea con Comportamiento Humano** — encuestas ambas-positivas, easter eggs, keywords

### 8.2 Los 4 patrones rítmicos

- **V Profunda** (superación): ↑→↓↓↓→↑→↓↓↓→↑↑↑↑↑
- **Zigzag** (relaciones): →↑↓↑↓↑↓↑↓↑
- **Gran Ola** (amor/pérdida): ↑↑↑↑↑→↓↓→↑↑↑↑↑↑↑→↓↓↓→→→→
- **Escalera** (autoridad): ↑↑↑↑↑↑↑↑↑↑↑↑

**Regla de los 3:** NUNCA más de 3 slides DOWN consecutivas. 100% consistente en 88 guiones.

### 8.3 Las 13 ausencias deliberadas

Naffe NUNCA: vende en el primer tercio | muestra producto por dentro | justifica precio | menciona competidores | usa voz en off | pide permiso para vender | usa culpa/desesperación | usa "link en bio" | usa descuentos | publica story aislada | graba en estudio neutro | hace meta-contenido | usa texto largo en pantalla.

### 8.4 Overlay ≠ hablado (Naffe) → H48 (Javier)

**Naffe (base):** En stories con video, el overlay es un TÍTULO corto. El hablado EXPANDE con emoción. Son capas complementarias. El viewer sin sonido se lleva la idea, con sonido la emoción.

**H48 (evolución, §2.8):** Javier lleva esto al extremo: el overlay NO es título del audio sino un PRODUCTO INDEPENDIENTE (poster motivacional) y el audio es OTRO PRODUCTO INDEPENDIENTE (confesión personal). No son complementarios — son 2 experiencias completas para 2 audiencias. La diferencia con Naffe: Naffe expande (mismo tema, más profundidad), H48 contradice (temas diferentes o incluso opuestos). Naffe usa TH en 2-3 slides de 10. H48 usa stories ENTERAS de 30-60s.

### 8.5 Orden de apilamiento de autoridad

Experiencial PRIMERO (dolor, esfuerzo) → Prestada SEGUNDO (Columbia, ONU) → Social TERCERO (números) → Demostrada al FINAL (enseñar). Mostrar dolor antes de credenciales = logro ganado. Credenciales sin dolor = showoff.

### 8.6 Evolución del sistema (7 meses)

Ago: stories épicas 15+ slides → Feb: secuencias eficientes 5 slides. El ratio buildup:CTA pasó de 88:12 a 60:40. Keywords migraron a links. Scripts pasaron de literales a instrucciones para equipo. La versión más reciente es más corta, más directa y está diseñada para escalar.
